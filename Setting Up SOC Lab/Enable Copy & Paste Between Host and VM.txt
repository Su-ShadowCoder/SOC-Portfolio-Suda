========================================================
Enable Copy & Paste Between Host and VM
Most Direct and Easiest Method
PROCEEDING MANUAL
========================================================

SCOPE
-----
Primary focus: Linux (Ubuntu) guest on VMware
Secondary reference: Windows guest (brief)

--------------------------------------------------------
LINUX (UBUNTU) GUEST ON VMWARE (WORKSTATION / PLAYER)
--------------------------------------------------------

GOAL
- Enable bidirectional clipboard (Host ↔ VM)
- Enable drag & drop

PREREQUISITES
- VMware Workstation or VMware Player installed on host
- Ubuntu desktop installed inside VM
- VM powered ON (not suspended)

STEP 1 — INSTALL VMWARE TOOLS (OPEN SOURCE VERSION)
--------------------------------------------------
Inside the Ubuntu VM, open a terminal and run:

sudo apt update
sudo apt install -y open-vm-tools open-vm-tools-desktop
sudo reboot

STEP 2 — VERIFY SERVICES ARE RUNNING
------------------------------------
After reboot, open terminal:

systemctl status vmtoolsd

EXPECTED RESULT:
- Status: active (running)

If NOT active, run:

sudo systemctl enable --now vmtoolsd

STEP 3 — VERIFY DESKTOP INTEGRATION PACKAGE
-------------------------------------------
Run:

dpkg -l | grep open-vm-tools

EXPECTED RESULT:
- open-vm-tools
- open-vm-tools-desktop

STEP 4 — ENABLE CLIPBOARD ON HOST SIDE
--------------------------------------
On the HOST machine:

1. Power OFF the VM
2. Open VM Settings
3. Go to:
   Options → Guest Isolation
4. ENABLE:
   - Enable copy and paste
   - Enable drag and drop
5. Start the VM

STEP 5 — CONFIRM SESSION TYPE (IMPORTANT)
-----------------------------------------
Clipboard works best on X11.

Inside Ubuntu VM:

echo $XDG_SESSION_TYPE

EXPECTED:
- x11

If output is "wayland":
- Log out
- At login screen, select:
  "Ubuntu on Xorg"
- Log in again

STEP 6 — TEST
-------------
1. Copy text from host
2. Paste into Ubuntu VM terminal or text editor
3. Copy text from VM
4. Paste into host

SUCCESS INDICATORS
------------------
- Clipboard works both directions
- Drag & drop works
- vmtoolsd service running

--------------------------------------------------------
WINDOWS GUEST ON VMWARE (REFERENCE ONLY)
--------------------------------------------------------

WHAT I WOULD DO (DIRECT METHOD)
-------------------------------
1. Power ON Windows VM
2. VMware menu:
   VM → Install VMware Tools
3. Inside Windows:
   - Run installer
   - Choose Typical install
   - Reboot when prompted

HOST SIDE CHECK
---------------
VM Settings → Options → Guest Isolation
- Enable copy and paste
- Enable drag and drop

RESULT
------
Clipboard and drag & drop work automatically after reboot.

--------------------------------------------------------
IF CLIPBOARD STILL FAILS (FALLBACK METHODS)
--------------------------------------------------------

OPTION 1 — SHARED FOLDERS (MOST RELIABLE)
-----------------------------------------
VM Settings → Options → Shared Folders
- Enable
- Set folder path
- Access inside VM:
  /mnt/hgfs/

OPTION 2 — SCP (SECURITY LAB FRIENDLY)
--------------------------------------
scp file.txt user@host:/path

--------------------------------------------------------
SECURITY NOTE (SOC / LAB CONTEXT)
--------------------------------------------------------
- Clipboard sharing may be intentionally blocked
- This is normal in hardened or malware labs
- Shared folders or SCP are preferred in such cases
